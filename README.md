# 📨 Telegram Auto-Sender (Mass Invite & Message)

Скрипт для автоматического вступления в чаты/каналы по списку ссылок и отправки заранее подготовленного сообщения (с фото или без).  

Использует **Telethon** и ваш личный аккаунт для обхода ограничений ботов. Работает через MTProto-прокси, поддерживает паузы между действиями и циклический запуск.

## 📌 Возможности

- 🔗 **Вступление по ссылкам** – поддерживает публичные каналы (`@username`) и приватные чаты по инвайт-ссылкам (`t.me/+hash`).
- 📝 **Отправка кастомного сообщения** – текст загружается из файла `template.txt`, поддерживает подстановку имени пользователя (`{username}`).
- 🖼️ **Поддержка фото** – если в папке есть `photo.png`, он будет прикреплён к каждому сообщению.
- 🔁 **Циклический запуск** – после обработки всех ссылок скрипт ждёт 15 минут и повторяет цикл (можно отключить).
- 🌐 **MTProto-прокси** – поддержка прокси с секретом для обхода блокировок.
- ⏱️ **Интеллектуальные паузы** – автоматическое ожидание при flood-ограничениях, а также пауза 50 секунд между чатами.
- 📋 **Обработка ошибок** – логирование проблем с вступлением, отправкой, недействительными ссылками.

## 🛠 Технологии

- **Python 3.10+**
- **Telethon** – работа с Telegram API от лица пользователя
- **dotenv** – управление переменными окружения
- **asyncio** – асинхронное выполнение

## 🔐 Настройка переменных окружения (`.env`)

```env
# Данные аккаунта Telegram
number = '+71234567890'          # Номер телефона (с кодом страны)
api_id = 1234567                 # Получить на my.telegram.org
api_hash = 'ваш_api_hash'        # Получить на my.telegram.org

# ---------- НАСТРОЙКИ MTProto-ПРОКСИ ----------
PROXY_HOST = 'proxy.example.com'
PROXY_PORT = 1080
PROXY_SECRET = 'deadbeef...'      # Hex-строка секрета прокси
```

Важно: если прокси не используется, удалите соответствующие строки или закомментируйте.

```bash
pip install telethon dotenv
```

Запуск скрипта

```bash

```
При первом запуске Telethon запросит код подтверждения (придёт в Telegram). Введите его в консоль.

## 🧠 Логика работы

Скрипт подключается к Telegram через ваш аккаунт (с использованием прокси, если указан).

Читает список ссылок из users.txt.

Для каждой ссылки:

Пытается вступить в чат/канал (поддерживаются публичные username и приватные инвайт-ссылки).

Получает сущность чата.

Загружает текст из template.txt и подставляет {username} (если есть).

Отправляет сообщение (с фото, если есть photo.png).

Ждёт 50 секунд перед следующим чатом.

После обработки всех ссылок – ждёт 15 минут и повторяет цикл (бесконечно, до остановки Ctrl+C).

## ⚙️ Настройка параметров

В коде вы можете изменить:

Паузу между чатами – await asyncio.sleep(50) (строка ~87)

Интервал между циклами – await asyncio.sleep(900) (строка ~99) – 15 минут.

Имена файлов – TEMPLATE_FILE, USERS_FILE, PHOTO_FILE (начало файла).

## ⚠️ Важные замечания

Аккаунт должен быть в хорошем репутационном статусе – новые аккаунты часто получают спам-блоки.

Flood-лимиты – если вы отправляете много сообщений, Telegram может временно ограничить действия. Скрипт умеет ждать, но это не панацея.

Прокси – использование прокси обязательно в регионах с блокировкой Telegram. Убедитесь, что секрет прокси указан корректно (hex-строка).

Файлы должны быть в кодировке UTF-8 – для корректного чтения текста.

Запуск от имени пользователя – скрипт использует ваш личный аккаунт, а не бота. Никогда не передавайте свой номер, api_id и api_hash третьим лицам.
